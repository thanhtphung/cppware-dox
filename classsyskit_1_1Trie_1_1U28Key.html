Detailed Description

unsigned 28-bit key

A class representing an unsigned 28-bit key. It helps form normalized keys used by a digit trie. This implementation breaks a 28-bit number into a sequence of 7 bytes with each byte holding 4 bits. A 28-bit number resides in the least significant 28 bits of an unsigned int. For example, the normalized form of 0x01234567UL is the 07-01-02-03-04-05-06-07 hexadecimal byte sequence. Construct an instance or use encode() to normalize a native key. Use decode() to decode a normalized key. Example:

Trie::U28Key key(k);
assert(Trie::U28Key::decode(key) == k);
