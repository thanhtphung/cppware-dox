26 class Cmd
27  //!
28  //! A class representing a command handler. The onRun() method is invoked when
29  //! a command is run. The method should return true/false to indicate command
30  //! validation success/failure. A failure will result in a command usage hint.
31  //! The method is invoked in a dedicated agent task. Each Cmd instance is
32  //! associated w/ one or more commands (e.g., one Cmd class can be used to
33  //! implement both "task-suspend" and "task-resume"), and each command is
34  //! associated w/ at most one Cmd instance. The association is specified when
35  //! a Cmd instance is constructed. Command outputs are required and must be
36  //! generated using the formRsp() and/or respond() methods. Multiple onRun()
37  //! invocations can occur concurrently.
38  //!
