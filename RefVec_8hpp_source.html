16 //! vector of reference-counted items
17 class RefVec: private Vec
18  //!
19  //! A class representing an unsorted vector of reference-counted items. The
20  //! vector has an initial capacity of capacity() items. The capacity can
21  //! grow as needed if the growth factor is set to non-zero when constructed
22  //! or afterwards using setGrowth(). If the vector is growable, growth can
23  //! occur when items are added. Items are added using various forms of
24  //! the add() method and removed using various forms of the rm() method.
25  //! Updates can be expensive if many items need to be shifted up or down.
26  //! For best update performance, items should be added to the tail end,
27  //! and items should be removed without maintaining the current ordering.
28  //! In general, upon vector entrance, an item's reference count is incremented;
29  //! and upon vector exit, the item's reference count is decremented. Example:
30  //!\code
31  //! RefVec vec;
32  //! vec.add(item0);
33  //! vec.addAtIndex(1 /*index*/, item1);
34  //! vec.rm(item2, false /*maintainOrder*/);
35  //! vec.rm(item3, true /*maintainOrder*/);
36  //! vec.rmFromIndex(1 /*index*/, false /*maintainOrder*/);
37  //! vec.rmFromIndex(0 /*index*/, true /*maintainOrder*/);
38  //!\endcode
39  //!
