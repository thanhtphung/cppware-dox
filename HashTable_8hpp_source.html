17 //! hash table of opaque items
18 class HashTable: public Growable
19  //!
20  //! A class representing a hash table of opaque items. Chaining is used
21  //! to resolve hash collisions. Items should be unique. If they are not,
22  //! a successful search would return any matching item. Items are added
23  //! using the add() method and removed using the rm() method. Searches are
24  //! provided by the find() methods. A hash table has zero or more buckets,
25  //! and each bucket holds zero or more items. The utilized hash function
26  //! dictates which bucket an item must reside in. Searching of items
27  //! residing in the same bucket requires a linear search within the bucket.
28  //! Best performance is achieved if the hash function is efficient and if
29  //! the hash function distributes the items evenly in the available buckets
30  //! and if the average itemsPerBucket ratio is reasonably small. It's
31  //! assumed a simple modular hash function is used. The number of buckets
32  //! remains a prime number when buckets are added to avoid an undesirable
33  //! itemsPerBucket ratio.
34  //!
35 {
