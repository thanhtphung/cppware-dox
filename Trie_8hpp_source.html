17 //! digit trie
18 class Trie
19  //!
20  //! A class representing a digit trie of non-zero opaque items. A digit trie can
21  //! serve as a map of digit strings (keys) to non-zero opaque items (values). A
22  //! key is represented as a contiguous array of digits prefixed by its length. A
23  //! key is valid for a trie if it has at least one digit and if its digits are
24  //! within the trie's digit range. The digit range is specified at construction
25  //! and cannot be changed. A few Trie::XxxKey classes are provided to help
26  //! form normalized keys. A typical trie uses direct indexing to locate children
27  //! nodes. This implementation uses direct indexing only when a node has more than
28  //! 5 children. A linear search is used when a node has less than 5 children.
29  //! Example:
30  //!\code
31  //! Trie trie;
32  //! Trie::U32Key k(123);
33  //! trie.add(k, (void*)(0xabc));
34  //! void* foundV;
35  //! if ((! trie.find(k, foundV)) || (foundV != (void*)(0xabc)))
36  //! {
37  //! assert("coding error?" == 0);
38  //! }
39  //!\endcode
40  //!
41 {

47  MaxKeyLength = 255
